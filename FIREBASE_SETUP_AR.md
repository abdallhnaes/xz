# إعداد Firebase لتطبيق زاجل

هذه النسخة تستخدم Firebase Realtime Database للمزامنة بين الأجهزة.

## خطوات الإعداد

1. افتح Firebase Console.
2. ادخل إلى مشروعك.
3. افتح Realtime Database.
4. أنشئ قاعدة البيانات إذا لم تكن موجودة.
5. افتح تبويب Rules.
6. انسخ محتوى ملف `database.rules.json`.
7. الصقه في Rules ثم اضغط Publish.

## ملف إعداد Firebase داخل التطبيق

افتح الملف:

`app/src/main/assets/zajel_bus_app/firebase-config.js`

إذا أنشأت مشروع Firebase جديدًا، استبدل القيم من:

Project settings > General > Your apps > Web app config

## مسار المزامنة

التطبيق يزامن البيانات في Firebase داخل المسار:

`zajel_secure_apps / zajel_main`

يمكن تغيير `zajel_main` من ملف `firebase-config.js` عند الحاجة.

## نظام الحماية

- رمز المالك لا يظهر داخل التطبيق ولا في الإعدادات.
- رمز المالك يطلب مرة واحدة فقط بعد تثبيت التطبيق على الجهاز.
- المدير والموظف لكل منهما PIN خاص.
- كل حساب يستطيع تغيير PIN الخاص به بنفسه.
- المدير يستطيع إعادة تعيين PIN الموظف دون رؤية الرمز القديم.
- الموظف لا يستطيع التعديل على الفواتير أو الإعدادات.

## إشعارات الهاتف

تمت إضافة إشعارات Android محلية مرتبطة بتغييرات Firebase داخل التطبيق:

- تعديل فاتورة.
- حذف فاتورة.
- تعديل إعدادات.
- إشعار يدوي من المدير.
- رسالة من الموظف للمدير.
- رد المدير للموظف.

ملاحظة:
للإشعارات التي تصل حتى لو كان التطبيق مغلقًا بالكامل، نحتاج Firebase Cloud Messaging وإعدادًا إضافيًا.
